## Key Features

### Raw Output Panel
- **File tree view** with expandable folders
- **Inline preview** for JSON, CSV, MD, images
- **Syntax highlighting** for code files
- **Copy to clipboard** for any content
- **Download individual files** or all as ZIP

### Generated Visualization Panel
- **Iframe sandbox** renders the AI-generated HTML/JS
- **Full-screen mode** for detailed exploration
- **Download HTML** to use elsewhere
- **Shareable link** generates a hosted version

# Option B: Tabbed Full-Width View

## Concept

A tabbed interface where users switch between different views of the outputs. The visualization takes full width for maximum impact, with raw outputs accessible via tabs.

# Agent 2: UI Generator - System Prompt

```python
UI_GENERATOR_SYSTEM_PROMPT = """
You are a UI visualization expert. Your task is to create beautiful,
interactive HTML + vanilla JavaScript dashboards to visualize data outputs.

## Your Inputs
1. ORIGINAL USER PROMPT - What the user was trying to accomplish
2. OUTPUT FILES - All files produced by the task agent

## Your Task
Create a single index.html file that:
1. Beautifully visualizes ALL relevant output data
2. Is tailored to what the user wanted to achieve
3. Uses modern, clean design (think: Apple, Linear, Notion)
4. Is fully self-contained (inline CSS, vanilla JS, no external deps except CDNs)
5. Is responsive and works on mobile

## Design Principles
- Visual hierarchy: Most important insights first
- Use charts/graphs for numerical data (use Chart.js from CDN)
- Use tables for detailed data (sortable, searchable)
- Use cards for key metrics
- Use color meaningfully (red=bad, green=good, etc.)
- Include the raw data in expandable sections
- Add download buttons for data export

## Technical Requirements
- Single index.html file with inline <style> and <script>
- Load Chart.js from CDN if needed: https://cdn.jsdelivr.net/npm/chart.js
- Embed data directly in JavaScript (no fetch calls)
- Make charts interactive (hover tooltips, click to filter)
- Include a "View Raw Data" toggle

## Output Format
Save your visualization as: output/visualization/index.html

## Examples of Good Visualizations

For SALES DATA:
- Hero metrics: Total revenue, growth %, top product
- Line chart: Revenue over time
- Bar chart: Product comparison
- Table: Detailed transactions

For FEEDBACK ANALYSIS:
- Sentiment pie chart
- Word cloud of common terms
- Timeline of sentiment changes
- Categorized issue cards
- Quote highlights

For COMPARISON DATA:
- Side-by-side comparison cards
- Difference highlighting (green/red)
- Winner badges
- Detailed comparison table

For RESEARCH RESULTS:
- Key findings cards
- Source citations
- Related links
- Summary paragraphs

Remember: You're creating this for visual learners. Make the data BEAUTIFUL
and IMMEDIATELY understandable.
"""
```

# Implementation Architecture

```python
# Main execution flow

async def run_agent_with_visualization(
    user_prompt: str,
    user_files: dict[str, bytes],
) -> AgentResult:

    # === AGENT 1: Task Executor ===
    async with Evolve(
        config=AgentConfig(type="claude", api_key=EVOLVE_KEY),
        workspace_mode="knowledge",
    ) as executor:
        # Upload user's files
        await executor.upload_context(user_files)

        # Run the main task
        result = await executor.run(prompt=user_prompt)

        # Get all output files
        raw_output = await executor.get_output_files(recursive=True)

    # === AGENT 2: UI Generator ===
    async with Evolve(
        config=AgentConfig(type="claude", api_key=EVOLVE_KEY),
        workspace_mode="knowledge",
        system_prompt=UI_GENERATOR_SYSTEM_PROMPT,
    ) as visualizer:
        # Provide context: original prompt + output files
        context = {
            "original_prompt.txt": user_prompt,
            **{f"outputs/{k}": v for k, v in raw_output.files.items()}
        }
        await visualizer.upload_context(context)

        # Generate the visualization
        await visualizer.run(
            prompt="""
            Create a beautiful HTML visualization for the outputs in the
            context folder. The original user prompt is in original_prompt.txt.
            The task outputs are in the outputs/ folder.

            Save your visualization to: output/visualization/index.html
            """
        )

        visualization_output = await visualizer.get_output_files(recursive=True)

    return AgentResult(
        raw_files=raw_output.files,
        visualization_html=visualization_output.files.get("visualization/index.html"),
        execution_log=result.stdout,
    )
```

# Comparison: Option A vs Option B

| Aspect | Option A: Split View | Option B: Tabbed View |
|--------|---------------------|----------------------|
| **Screen Usage** | 50/50 split | Full width per view |
| **Comparison** | Easy to compare raw vs viz | Need to switch tabs |
| **Focus** | Good for technical users | Better for visual learners |
| **Mobile** | Stacks vertically | Works natively |
| **Complexity** | Slightly more complex | Simpler to implement |
| **Best For** | Power users, debugging | Presentation, sharing |

---

# Recommended: Start with Option B

**Reasons:**
1. **Simpler to build** - Single view at a time
2. **Better mobile experience** - No awkward splits
3. **Visualization gets full attention** - The "wow" factor
4. **Easier to share** - Screenshot/share just the viz
5. **Follow-up tab enables iteration** - Users can refine

**Then iterate:**
- Add Option A as a "Developer View" toggle
- Let users customize their preferred layout
- Add more view modes (Presentation, Print, etc.)
